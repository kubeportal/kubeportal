KubePortal

The landing page for Kubernetes clusters.

KubePortal is a web application with two parts:
The frontend acts as landing page for your Kubernetes installation. It implements single sign-on for the portal itself and other web applications, such as Kubernetes Dashboard and Grafana. Users can request cluster access and download their kubectl config files after they got accepted.
The backend enables web-based user management for the cluster. Portal users can be mapped to Kubernetes namespaces resp. service accounts. Requests for access can be accepted / rejected by multiple administrators. The frontend page can be customized.

The KubePortal functionality can also be achieved with a cascade of OAuth2 proxies, an LDAP server with schema extension, an LDAP UI and some stunts the get the service accounts being created. It is basically the all-inclusive version of Dex.

Single Sign-On

Users can use their organizational LDAP / Active Directory login or even social logins, when enabled by the administrators.

New users

Users can request cluster access with a single click.

Config file

After acceptance by the admins, a cluster user can download its kubectl config file directly from the portal.

Portal links

Administrators can configure quick links for web applications. URL placeholders for namespace and account are supported, so Kubernetes Dashboard users can be directly forwarded to their namespace overview.

Full grading freedom

Gradings can be delayed and commented. When validation with test scripts is enabled, the corrector sees all the script output and can decide on the result.

Single information source

The grading table gives a summary of all students and their gradings. It can be download for further processing with Excel or other tools.

Dealing with large courses

Submissions can be filtered and downloaded for offline correction. There are also support functions for re-validation and bulk emails.

Faster corrections

Student submission archives (ZIP / TGZ) can be previewed on the web page. Downloading all submissions is also supported.

Automated testing of submissions

OpenSubmit can take a student upload and extract it on separate test machines. Each assignment has its own set of test scripts. Validation tests run, visibly for the student, before the deadline. When the deadline is over, the system can run another invisible full test for the correctors. Test machines report their software configuration in the student dashboard.
